Clase 1: Relación entre clases y objetos en un sistema

Introduccion

En la programación orientada a objetos (POO), las clases y los objetos son los pilares fundamentales. Una clase es un molde o plantilla que define atributos y métodos, mientras que un objeto es una instancia concreta de esa clase.

🔹 Ejemplo

        
Java

class Libro {
    String titulo;
    String autor;
    int paginas;

    // Constructor
    Libro(String titulo, String autor, int paginas) {
        this.titulo = titulo;
        this.autor = autor;
        this.paginas = paginas;
    }

    // Método
    void mostrarInfo() {
        System.out.println("Título: " + titulo + 
                           ", Autor: " + autor + 
                           ", Páginas: " + paginas);
    }
}

public class Main {
    public static void main(String[] args) {
        Libro libro1 = new Libro("Cien años de soledad", "Gabriel García Márquez", 471);
        Libro libro2 = new Libro("El Principito", "Antoine de Saint-Exupéry", 96);

        libro1.mostrarInfo();
        libro2.mostrarInfo();
    }
}

🔹 Explicación

La clase es abstracta y no ocupa memoria hasta que se crea un objeto. El objeto es tangible dentro del programa, ocupa memoria y puede interactuar con otros. La relación es de molde → producto: la clase es el diseño, el objeto es la construcción.

🔍 Explicación adicional

this: es una palabra clave en Java que hace referencia al objeto actual. Dentro del constructor, this.titulo se refiere al atributo de la clase, mientras que titulo (sin this) es el parámetro recibido. Esto evita confusiones y asegura que el valor del parámetro se asigne correctamente al atributo del objeto.

new Libro(...): la palabra clave new crea una nueva instancia de la clase Libro. Cada vez que usamos new Libro("texto", "autor", páginas), se reserva memoria y se ejecuta el constructor para inicializar los atributos. Así obtenemos un objeto único con sus propios valores, independiente de otros objetos de la misma clase.

🔹 Ejercicio a resolver

Crea una clase llamada Estudiante con atributos nombre, edad y carrera. Define un constructor, crea dos objetos y muestra su información con un método mostrarInfo().
👉 Reto adicional: agrega un método que calcule si el estudiante es mayor de edad.

🔹 Conclusión

La relación entre clases y objetos es esencial en la POO: las clases son el diseño conceptual y los objetos son las instancias que hacen tangible ese diseño. Gracias a esta relación, los sistemas pueden ser más organizados, escalables y fáciles de mantener.

codigo completo: 
        
Java

      // Definición de la clase Estudiante
class Estudiante {
    // Atributos de la clase (propiedades de cada estudiante)
    String nombre;
    int edad;
    String carrera;

    // Constructor: se ejecuta al crear un nuevo objeto Estudiante
    // Recibe parámetros y asigna valores a los atributos usando 'this'
    Estudiante(String nombre, int edad, String carrera) {
        this.nombre = nombre;   // 'this.nombre' se refiere al atributo de la clase
        this.edad = edad;       // 'edad' es el parámetro, 'this.edad' es el atributo
        this.carrera = carrera; // asignamos el valor recibido al atributo
    }

    // Método para mostrar la información del estudiante
    void mostrarInfo() {
        System.out.println("Nombre: " + nombre +
                           ", Edad: " + edad +
                           ", Carrera: " + carrera);
    }

    // Método adicional: verifica si el estudiante es mayor de edad
    boolean esMayorDeEdad() {
        return edad >= 18; // devuelve true si edad >= 18, false en caso contrario
    }
}

// Clase principal donde se ejecuta el programa
public class Main {
    public static void main(String[] args) {
        // Creamos dos objetos de tipo Estudiante usando 'new'
        Estudiante est1 = new Estudiante("Ana Pérez", 20, "Ingeniería");
        Estudiante est2 = new Estudiante("Luis Gómez", 17, "Medicina");

        // Mostramos la información del primer estudiante
        est1.mostrarInfo();
        // Llamamos al método esMayorDeEdad() para verificar si es mayor de edad
        System.out.println("¿Es mayor de edad? " + est1.esMayorDeEdad());

        // Mostramos la información del segundo estudiante
        est2.mostrarInfo();
        // Verificamos si el segundo estudiante es mayor de edad
        System.out.println("¿Es mayor de edad? " + est2.esMayorDeEdad());
    }
}
